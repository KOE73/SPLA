Портативный локальный AI-ассистент.

**Подключите** локальную LLM, **откройте** проект и начните **работать**.

## Типовые задачи

- Анализировать кодовую базу.
- Объяснять архитектуру проекта.
- Искать и изменять файлы.
- Запускать сборку и тесты.
- Диагностировать сетевые проблемы.
- Работать с предметными плагинами, например для 1C.

## Почему SPLA

- Локальный подход.
- Совместимость с OpenAI API.
- Ориентация на проекты.
- Расширение через плагины.
- Явная модель разрешений.

## Основные возможности

1. **Локальные LLM.** SPLA рассчитан на локальные модели и OpenAI-compatible API. Конфигурация по умолчанию - LM Studio по адресу `http://127.0.0.1:1234/v1`, но можно использовать другой совместимый runtime.
2. **Расширяемая система инструментов.** Инструменты регистрируются через MCP-хост и плагины.
3. **Организация проектов.** Проект SPLA описывается файлом `.spla` в рабочей директории. В нем задаются рабочая область, режим агента, инструкции, документация, игнорируемые пути, LLM endpoint, плагины и разрешения. Файл можно связать с приложением и открывать как отдельную рабочую область.
4. **Режимы безопасности.** Режимы `Chat`, `Research`, `Inspect`, `Edit` и `Agent` ограничивают чтение файлов, запись файлов, shell-команды, доступ к сети и автономность агента.

## Инструменты

- Базовые системные инструменты: чтение, запись, создание, удаление, patch, поиск файлов, поиск текста, выполнение команд, текущая дата и время, контекст, `dotnet build`, `dotnet test`, web fetch, web search.
- Сетевые инструменты: сведения о хосте, LAN scan, port scan, ping, nslookup, HTTP GET, HTTP HEAD, проверка порта, traceroute.

## Быстрый старт и сборка из исходников

Требования:
- для сборки: .NET 10.
- для запуска: локальный LLM runtime, например LM Studio, и .NET 10 Runtime.

1. Соберите desktop app, CLI, плагины и ZIP-пакет общим скриптом:

```powershell
.\PublishAll.cmd
```

Готовая папка приложения создается в `.publish/work/`. Ее можно перенести в другое место и запускать SPLA оттуда. ZIP-архив создается в `.publish/zip/SPLA.zip`.

2. Запустите LM Studio или другой локальный OpenAI-compatible endpoint либо убедитесь, что у вас есть доступ к нему.
3. Запустите desktop UI из `.publish/work/` и укажите путь к вашему LLM endpoint.

Также можно запустить CLI-версию из `.publish/work/`, чтобы работать через терминал.

Экран настроек содержит краткое описание. Проект можно создать в любой папке. На практике это файл `*.spla` с настройками, который можно открыть через UI или CLI. Проект определяет разрешения, инструкции, документацию, игнорируемые пути, плагины и связанные настройки.

Одно окно означает один проект. Если нужно работать с несколькими проектами, откройте несколько окон. Проекты изолированы друг от друга, но могут использовать одни и те же плагины и инструменты.

## Безопасность

SPLA использует 5 режимов агента:

| Режим | Назначение |
|--------|--------|
| Chat | Только обсуждение |
| Research | Чтение и анализ |
| Inspect | Диагностика и инспекция |
| Edit | Изменение файлов проекта |
| Agent | Автономное многошаговое выполнение |

## Плагины

Плагины располагаются рядом с опубликованным приложением в директории `plugins/`. У каждого плагина есть собственный `meta.yaml`; плагин может добавлять инструменты, prompt-инструкции и, при необходимости, Avalonia UI-панели. Правила именования инструментов и metadata плагинов описаны в [plugins.md](agents/plugins.md).

## Ответственное использование

Сетевые инструменты SPLA предназначены для исследований, диагностики, инвентаризации и обычной работы сетевых администраторов с системами и сетями, которыми они владеют, управляют или которые им явно разрешено проверять.

Не используйте SPLA для несанкционированного сканирования, probing, попыток доступа, нарушения работы или любых действий против сторонних сетей без разрешения.
