# SPLA - локальный AI-ассистент

  <img src="Images/MainLogo.png" alt="SPLA Logo" width="300" />
</p>


SPLA - локальный AI-ассистент для работы с файлами, проектами, инструментами ОС и предметными плагинами через локальные LLM runtime. 

Большинство AI-агентов требуют установки расширений, настройки окружения, конфигурации инструментов, Docker, MCP-серверов или сложной интеграции с IDE.

SPLA ориентирован на другой подход:

0. Иметь локальный LLM runtime, такой как LM Studio, или OpenAI-compatible API.
1. Скачать архив.
2. Запустить SPLA.
3. Указать OpenAI-compatible API endpoint.
4. Начать работу.

- Без IDE.
- Без Docker.
- Без браузера.
- Без облака.
- Без регистрации.

Все работает локально из одной папки.

SPLA можно использовать из CLI или через desktop UI на Avalonia.

## Основные возможности

1. **Локальные LLM.** SPLA ориентирован на локальные модели и OpenAI-compatible API. Базовый вариант - LM Studio по адресу `http://127.0.0.1:1234/v1`, но можно подключать другой совместимый runtime.
2. **Расширяемая система инструментов.** Инструменты подключаются через MCP-хост и плагины. 
3. **Организация проектов.** Проект SPLA описывается `.spla`-файлом в рабочей папке. В нем задаются workspace, режим агента, инструкции, документация, ignored paths, LLM endpoint, плагины и разрешения. Такой файл можно ассоциировать с приложением и открывать проект как отдельное рабочее пространство.
4. **Режимы безопасности.** Режимы `Chat`, `Research`, `Inspect`, `Edit`, `Agent` ограничивают чтение, запись, shell-команды, сеть и степень автономности агента.

## Инструменты

- Базовые системные tools: чтение, запись, создание, удаление, patch, поиск файлов, поиск текста, выполнение команд, текущая дата/время, контекст, `dotnet build`, `dotnet test`, web fetch, web search. 
- Сетевые tools: сведения о хосте, LAN scan, port scan, ping, nslookup, HTTP GET, HTTP HEAD, проверка порта, traceroute. 
- В русской версии отдельно важен 1C-плагин: индексирование конфигурации, поиск и объяснение объектов, зависимости, обратные зависимости, ссылки, readers/writers и графы.

## Быстрый старт и публикация

Требования:
- для сборки: NET 10.
- для запуска: локальный LLM runtime (например, LM Studio) и .NET 10 Runtime.

1. Соберите desktop app, CLI, плагины и ZIP-пакет общим скриптом:

```powershell
.\PublishAll.cmd
```
Готовая папка приложения находится в `.publish/work/`. Ее можно перенести в другое место и пользоваться оттуда. ZIP-архив создается в `.publish/zip/SPLA.zip`.

2. Запустите LM Studio или другой локальный OpenAI-compatible endpoint или просто имейте доступ к такому.
3. Запустите desktop UI из `.publish/work/` и укажите путь к вашему LLM endpoint.

Можно также запустить CLI-версию из `.publish/work/` для взаимодействия через терминал.
 
В настройках есть краткое описание. Можно создать проект в любой папке. Это по сути фал `*.spla` с настройками, который можно открыть через UI или CLI. В проекте задаются разрешения, инструкции, документация, ignored paths, плагины и т.д.

Одно окно, один проект. Если нужно работать с несколькими проектами, можно открыть несколько окон. Проекты изолированы друг от друга, но при этом могут использовать одни и те же плагины и инструменты.


## Режимы безопасности

SPLA использует несколько режимов работы агента. Режим определяет, какие действия разрешены в рамках текущего проекта.

| Режим        | Назначение                                                                          |
| ------------ | ----------------------------------------------------------------------------------- |
| **Chat**     | Только ответы и обсуждение. Без доступа к файлам и инструментам.                    |
| **Research** | Чтение и анализ файлов, документации и проекта без изменений.                       |
| **Inspect**  | Диагностика проекта, системы и сети без внесения изменений.                         |
| **Edit**     | Изменение файлов и выполнение задач разработки в рабочей области проекта.           |
| **Agent**    | Автономное выполнение многошаговых задач с использованием разрешенных инструментов. |

Подробное описание режимов, разрешений и ограничений приведено в [readme_security_ru.md](docs/readme_security_ru.md).


## Плагины

Плагины располагаются рядом с опубликованным приложением в директории `plugins/`. Каждый plugin имеет собственный `meta.yaml`, может добавлять tools, prompt-инструкции и, при необходимости, Avalonia UI-панели. Правила именования tools и формат plugin metadata описаны в [plugins.md](agents/plugins.md).
