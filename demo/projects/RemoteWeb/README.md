# RemoteWeb

Демо управления внешним сайтом через встроенную панель Browser Lab в удалённом Web-клиенте SPLA.

- [RemoteWeb.spla](RemoteWeb.spla) включает только `browser_screencast` и пять команд, нужных агенту
  для навигации, просмотра элементов, клика, видимого посимвольного ввода и чтения текста.
- Обычный Browser-плагин, файловые и системные действия, skills и остальные плагины выключены.
- `Run.cmd` — ASCII-обёртка для запуска `Run.ps1` двойным кликом или из обычной командной строки.
- `Run.ps1` собирает Screencast-плагин, запускает Web-сервис на порту `5050`, открывает чат рядом
  с Browser Lab и передаёт стартовое задание через `spla serve --new-chat`.

После подключения агент откроет английскую Wikipedia во встроенной панели, найдёт строку поиска,
видимо напечатает `OpenAI`, перейдёт к статье и перескажет первый абзац в чате. Это одна и та же
страница: кадры видит человек, а команды Playwright выполняет агент.

Перед запуском поднимите LM Studio на этом компьютере по адресу `http://127.0.0.1:1234/v1` и
загрузите любую chat-модель.

Обычный запуск:

```cmd
Run.cmd
```

Прямой запуск из PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -File .\Run.ps1
```

Локально откроется `http://127.0.0.1:5050/?panel=browserScreencast`. С другого компьютера в
доверенной локальной сети откройте:

```text
http://<имя-компьютера>:5050/?panel=browserScreencast
```

CMD-файл содержит только ASCII и ничего не кодирует сам — он передаёт управление PowerShell.
PS1-скрипт явно настраивает ввод, вывод и передачу аргументов как UTF-8. Стартовое задание также
записано ASCII-текстом, поэтому одинаково передаётся в Windows PowerShell 5.1 и PowerShell 7.

> Демо слушает все сетевые интерфейсы без аутентификации и разрешает управление встроенным
> браузером. Запускайте его только в доверенной локальной сети и не публикуйте порт `5050`
> напрямую в интернет.
