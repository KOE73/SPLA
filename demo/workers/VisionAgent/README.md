# VisionAgent — автономный агент видеонаблюдения на SPLA Runtime

Демонстрация того, что `SPLA.Runtime` живёт без CLI, сервиса и UI: маленькое консольное
приложение подключает рантайм, указывает файл проекта — и получает полноценного агента.

## Что делает

Цикл: **кадр из видеоисточника → ход агента с картинкой → ответ модели в консоль → пауза → снова**.

- Источник: USB-камера (`usb:0`), RTSP-поток (`rtsp://…`) или видеофайл (путь).
- Промпт анализа — обычные `agent.instructions` проекта ([prompt.md](prompt.md)):
  время события, список горного оборудования в кадре, что оно делает, бортовой номер.
- Настройки демо — секция `vision:` в том же [vision.spla](vision.spla)
  (стандартный загрузчик SPLA незнакомые секции игнорирует).
- Контекст между кадрами по умолчанию не копится (`keep_history: 0`) — каждый кадр
  анализируется с чистого листа, окно локальной модели не переполняется.
- Ходы пишутся в обычный чат проекта — историю можно открыть любым клиентом SPLA.

## Запуск

1. Поднимите LM Studio с vision-моделью (например `google/gemma-4-12b-qat`):
   `lms server start && lms load google/gemma-4-12b-qat --context-length 8192 --yes`
2. Поправьте `vision.spla` (источник, endpoint, model).
3. ```
   dotnet run --project demo/workers/VisionAgent            # vision.spla берётся рядом
   dotnet run --project demo/workers/VisionAgent -- path/to/other.spla
   ```

Остановка — `Ctrl+C`. В конце всегда печатается явный статус: `готово` / `не готово`.

## Ключи секции `vision:`

| Ключ | Значение |
|------|----------|
| `source` | `usb:N` \| `rtsp://…` \| путь к файлу (относительно `.spla`) |
| `interval_seconds` | пауза между кадрами |
| `max_width` | ужать кадр до ширины перед отправкой (0 = как есть) |
| `jpeg_quality` | качество JPEG 1–100 |
| `keep_history` | сколько прошлых кадров-ходов держать в контексте |
| `save_frames` | папка для копий отправленных кадров (пусто = не сохранять) |
| `show_windows` | `true` = два окна предпросмотра: **Live** (непрерывный поток; для live-источников кадры читаются отдельным потоком и не замирают, пока модель думает) и **Sent to model** (кадр, только что ушедший на анализ). `false` = headless |
| `show_reasoning` | `true` = печатать рассуждения модели (chain-of-thought) серым цветом через Spectre.Console. На слабой модели их много — по умолчанию выключено |

После каждого кадра печатается строка токенов (голубым): вход/выход за этот ход
плюс накопительные суммы по проекту и по машине.

**Ускорение кадра.** Главный расход времени — размышления модели. Промпт их не отключает
(это режим самой модели); настоящий рычаг — `llm.reasoning_level` в `.spla`:
`off` выключает thinking у Gemma/Qwen полностью, `low/medium/high` — reasoning_effort
для gpt-oss-моделей. Плюс правило в промпте «по одному кадру движение не определить —
пиши "стоит"» убирает самый частый повод для долгих колебаний.
