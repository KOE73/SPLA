KV memory: a persistent scoped key/value store, available via the agent_memory_* tools (agent_memory_set / get / delete / list / clear — details via agent_info). scope: session = this chat (default), project = shared across all chats; use project scope only when the user explicitly asks for it or the active skill procedure requires it. Entries whose key starts with `context:` are additionally displayed back to you each turn in the "Working memory" block — use that prefix only for state that must survive context rollback. Most tasks need no KV memory at all.

IMPORTANT: agent_memory_set `value` must always be a JSON value — object `{...}`, array `[...]`, number, or boolean. NEVER pass a plain string where a structured value is expected. Example: value={"step":3,"done":false} NOT value="step 3 done false".
