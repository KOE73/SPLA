# Ограничения первой версии

Текущая реализация `SPLA.Plugins.OneC` является MVP (Minimum Viable Product) и имеет ряд осознанных ограничений.

## 1. Только файловая выгрузка
Индексатор работает только с конфигурацией, выгруженной в файлы (формат Конфигуратора или EDT). Нет прямого подключения к базе через COM-объекты или REST.

## 2. Эвристический парсер BSL
`OneCCodeAnalyzer` использует регулярные выражения, а не полноценное синтаксическое дерево (AST) языка 1С.
Возможны ложные срабатывания:
* Имена переменных, совпадающие с именами общих модулей.
* Нестандартно оформленные вызовы (многострочные и т.п.).
* Закомментированный код (парсер удаляет только простые `//` в строке, но не понимает сложные конструкции).

## 3. Ограниченный набор связей
Поддерживаются только основные типы связей (`owns`, `uses`, `calls`, `reads`, `writes`, `queries`). Динамические вызовы через `Выполнить()`, использование подписок на события, неявные вызовы через фоновые задания и другие сложные сценарии пока не отслеживаются.

## 4. Ограниченный парсинг запросов
`OneCQueryAnalyzer` ищет строковые литералы, начинающиеся с `ВЫБРАТЬ` или `SELECT`. Если запрос формируется динамически (конкатенация сложных строк) или хранится в макете, он может быть не распознан.

## 5. Нет LLM-построения индекса
На этапе построения индекса LLM не используется (для обеспечения скорости и детерминированности). Индекс строится строго алгоритмически.
