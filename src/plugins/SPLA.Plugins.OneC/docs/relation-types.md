# Типы связей (Relations)

Модуль `SPLA.Plugins.OneC` распознает несколько типов связей между объектами.
Для первой версии выбран минимальный, но наиболее ценный набор.

| Тип | Семантика | Источник (Confidence) |
|---|---|---|
| `owns` | Объект "владеет" другим объектом (например, Документ содержит Форму или ТабличнуюЧасть). | XML метаданные (`exact`) |
| `uses` | Объект упоминает другой объект (например, обращение к менеджеру справочника). | BSL код (`heuristic`) |
| `calls` | Модуль вызывает функцию ОбщегоМодуля. | BSL код (`heuristic`) |
| `reads` | Чтение данных из регистра или справочника. | BSL код (`heuristic`) |
| `writes` | Запись данных в регистр (например, через `Движения`). | BSL код (`heuristic`) |
| `queries` | Использование таблицы объекта в строке текста запроса. | BSL тексты запросов (`heuristic`) |

## Примеры

```text
Document.РеализацияТоваров
  --owns--> Form.ФормаДокумента
  --writes--> AccumulationRegister.ОстаткиТоваров
  --reads--> Catalog.Номенклатура
  --calls--> CommonModule.СкладскойУчет
  --queries--> AccumulationRegister.Продажи
```

## Эвристики BSL-анализатора

Парсер BSL работает на основе регулярных выражений, а не полноценного AST.
Это означает, что `calls`, `reads`, `writes`, `uses`, `queries` могут иметь ложные срабатывания (например, закомментированный нестандартным образом код, совпадение имен переменных с именами объектов). Поэтому им присваивается уровень уверенности `heuristic`.
