# Источники данных и структура выгрузки

Модуль `SPLA.Plugins.OneC` работает с файловой выгрузкой конфигурации 1С. Подключение к работающей базе по COM-объектам, OData или REST API в первой версии не поддерживается.

## Ожидаемая структура каталога

Конфигурация 1С может быть выгружена в файлы стандартными средствами платформы (Конфигуратор -> Выгрузить конфигурацию в файлы) или через утилиту `v8unpack` / EDT.

Мы ожидаем стандартную структуру, где каталоги верхнего уровня соответствуют типам метаданных:

```text
├── Documents/               # Документы
├── Catalogs/                # Справочники
├── AccumulationRegisters/   # Регистры накопления
├── InformationRegisters/    # Регистры сведений
├── AccountingRegisters/     # Регистры бухгалтерии
├── CommonModules/           # Общие модули
├── Reports/                 # Отчеты
├── DataProcessors/          # Обработки
├── Forms/                   # Общие формы
└── ...                      # И так далее
```

Внутри каталога метаданных каждый объект лежит в своей папке:

```text
Documents/
└── РеализацияТоваров/
    ├── РеализацияТоваров.mdo          <-- Описание объекта (XML)
    ├── Ext/
    │   ├── ObjectModule.bsl           <-- Модуль объекта
    │   └── ManagerModule.bsl          <-- Модуль менеджера
    └── Forms/
        └── ФормаДокумента/
            ├── ФормаДокумента.mdo     <-- Описание формы
            └── Ext/
                └── Form/
                    └── Module.bsl     <-- Модуль формы
```

## Как сканер (`OneCFileScanner`) понимает типы

1.  **Тип метаданных (Kind)** определяется по имени каталога верхнего уровня (например, `Documents/` -> `Document`).
2.  **Имя объекта (Name)** определяется по имени подкаталога (например, `РеализацияТоваров`).
3.  **Полное имя (FullName)** склеивается как `Kind.Name` (например, `Document.РеализацияТоваров`).

Файлы `.xml` и `.mdo` считаются метаданными и скармливаются `OneCMetadataParser`.
Файлы `.bsl` считаются исходным кодом и скармливаются `OneCCodeAnalyzer` и `OneCQueryAnalyzer`.

## Особенности форматов метаданных

Платформа 1С может выгружать XML-метаданные в разных форматах в зависимости от версии. `OneCMetadataParser` использует эвристики и XPath (через LINQ to XML), чтобы извлекать имена, синонимы и дочерние элементы вне зависимости от того, используется ли "толстый" формат до 8.3.10 или новый формат 1C:EDT с неймспейсами `mdclass:`.
