## Режимы безопасности

SPLA использует несколько уровней доступа. Режим определяет, какие инструменты и действия доступны агенту в рамках текущего проекта.

Это позволяет выбирать баланс между безопасностью и автономностью в зависимости от задачи.

| Режим        | Назначение                         | Чтение файлов | Изменение файлов | Команды ОС  | Сеть        | Автономность |
| ------------ | ---------------------------------- | ------------- | ---------------- | ----------- | ----------- | ------------ |
| **Chat**     | Ответы на вопросы и обсуждение     | Нет           | Нет              | Нет         | Нет         | Минимальная  |
| **Research** | Анализ документации и проекта      | Да            | Нет              | Нет         | Опционально | Низкая       |
| **Inspect**  | Диагностика и исследование системы | Да            | Нет              | Ограниченно | Да          | Средняя      |
| **Edit**     | Внесение изменений в проект        | Да            | Да               | Ограниченно | Да          | Средняя      |
| **Agent**    | Самостоятельное выполнение задач   | Да            | Да               | Да          | Да          | Максимальная |

### Chat

Наиболее безопасный режим.

Агент работает только как собеседник и не имеет доступа к файлам, инструментам, сети или операционной системе.

Подходит для обсуждения идей, архитектуры, планирования и консультаций.

---

### Research

Режим анализа.

Агент может читать файлы проекта, документацию и другие разрешенные источники данных, но не может изменять их.

Подходит для изучения существующего проекта, подготовки обзоров, поиска информации и объяснения кода.

Примеры задач:

* Объясни архитектуру проекта.
* Найди все реализации интерфейса.
* Составь обзор документации.
* Найди места использования класса.

---

### Inspect

Режим диагностики.

Агент получает доступ к диагностическим инструментам и может исследовать состояние системы, сети и проекта без внесения изменений.

Подходит для поиска проблем и анализа инфраструктуры.

Примеры задач:

* Проверь доступность сервера.
* Выполни ping и traceroute.
* Найди открытые порты.
* Проанализируй результаты сборки.

---

### Edit

Режим разработки.

Агент может изменять файлы внутри разрешенной рабочей области проекта.

Подходит для написания кода, исправления ошибок, обновления документации и рефакторинга.

Примеры задач:

* Исправь ошибку компиляции.
* Обнови документацию.
* Выполни рефакторинг класса.
* Создай новый модуль.

---

### Agent

Максимально автономный режим.

Агент может самостоятельно планировать последовательность действий, использовать инструменты, выполнять команды, анализировать результаты и продолжать работу до достижения поставленной цели в рамках разрешений проекта.

Подходит для сложных многошаговых задач.

Примеры задач:

* Найди и исправь проблему в проекте.
* Собери решение и устрани ошибки.
* Выполни анализ системы и подготовь отчет.
* Исследуй кодовую базу и реализуй новую функцию.
* Выполни задачу от начала до результата без пошагового участия пользователя.

Даже в режиме Agent агент ограничен настройками проекта, разрешенными инструментами, рабочей областью и политиками безопасности.
