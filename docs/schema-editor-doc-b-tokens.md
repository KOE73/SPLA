# Док Б — дизайн-токены (Avalonia ↔ Web)

Контракт Фазы 0 для **схемы №3** (стиль/тема). Отдельный мир от схем данных/UI (Док А) — не путать.

Цель: чтобы Web-рендеры (чат, JSON-редактор, возможно MD-редактор — все в WebView) и нативный
Avalonia-UI выглядели **похоже без жёсткой связи**. Достигается тем, что шарится **словарь токенов**
(имена + значения), а **не сам CSS/стиль**. У каждого рендера — своя привязка к этим именам.

---

## Принцип

- Схема данных/UI несёт **ноль визуала** (см. Док А). Весь визуал — здесь, в токенах.
- **Шарится список имён и значений**, не реализация:
  - Web биндит токен в **CSS-переменную** (`--color-accent`);
  - Avalonia биндит то же имя в **`DynamicResource`/`Style`**.
- Похожесть достигается **дисциплиной** (один дизайн-язык), а не общим кодом стилей.
  Попытка зашарить сам CSS = та жёсткая связь, которой не хотим (Avalonia его не съест).
- Это тот же принцип независимости рендереров, что и для чат-рендереров: общая семантика,
  раздельная реализация.

---

## Текущее состояние (зафиксировать на старте)

CSS для веб-чата уже формируется (значения сейчас, вероятно, зашиты в чат-рендере). Задача —
**вынести значения в этот общий словарь** и заставить чат-CSS ссылаться на токены, а не держать
их у себя. То есть словарь — единый источник, чат-CSS — потребитель.

## Формат словаря: таблица в три колонки

`имя токена | Avalonia-ресурс | Web CSS-переменная`. Имена — иерархические через точку
(`group.role.variant`). Значение — одно, привязок две.

### Цвета

| Токен | Avalonia (`DynamicResource`) | Web (CSS var) | Значение (пример) |
|-------|------------------------------|---------------|--------------------|
| `color.bg` | `Brush.Bg` | `--color-bg` | (из темы) |
| `color.surface` | `Brush.Surface` | `--color-surface` | |
| `color.text` | `Brush.Text` | `--color-text` | |
| `color.text.muted` | `Brush.TextMuted` | `--color-text-muted` | |
| `color.accent` | `Brush.Accent` | `--color-accent` | |
| `color.border` | `Brush.Border` | `--color-border` | |
| `color.danger` | `Brush.Danger` | `--color-danger` | |
| `color.ok` | `Brush.Ok` | `--color-ok` | |

### Отступы / размеры

| Токен | Avalonia | Web | Значение |
|-------|----------|-----|----------|
| `spacing.xs` | `Thickness.Xs` | `--spacing-xs` | 4 |
| `spacing.sm` | `Thickness.Sm` | `--spacing-sm` | 8 |
| `spacing.md` | `Thickness.Md` | `--spacing-md` | 12 |
| `spacing.lg` | `Thickness.Lg` | `--spacing-lg` | 20 |
| `radius.sm` | `CornerRadius.Sm` | `--radius-sm` | 4 |
| `radius.md` | `CornerRadius.Md` | `--radius-md` | 8 |

### Типографика

| Токен | Avalonia | Web | Значение |
|-------|----------|-----|----------|
| `font.body` | `FontFamily.Body` | `--font-body` | |
| `font.mono` | `FontFamily.Mono` | `--font-mono` | |
| `font.size.sm` | `FontSize.Sm` | `--font-size-sm` | 12 |
| `font.size.md` | `FontSize.Md` | `--font-size-md` | 14 |
| `font.size.lg` | `FontSize.Lg` | `--font-size-lg` | 18 |

> Значения — плейсхолдеры. Заполнить из существующей темы Avalonia + чат-CSS при инвентаризации.

---

## Темы

Словарь = набор **имён**. Значения зависят от темы (light/dark). Привязка одинаковая, значения разные:
- Web: `:root` / `[data-theme=dark]` переопределяют `--color-*`;
- Avalonia: `ThemeVariant` со своими ресурсами под теми же именами.

Имена токенов от темы **не зависят** — переключение темы не трогает разметку/схемы.

---

## Граница ответственности (чтобы не расползлось)

- Токен — **только** значение визуала (цвет/размер/шрифт). Никакой логики, никаких данных.
- Если тянет положить в токен что-то про *структуру* формы — это UI-схема (Док А), не токен.
- Если про *корректность данных* (enum, ограничение) — это схема данных (Док А), не токен.
