## SPLA: среда для агента, реестр моделей и автоматическое сравнение

SPLA — это среда, в которой основной агент получает цель, использует доменные инструменты и при необходимости запускает subagents через подходящие модели.

### Пример, с которого возникла идея

Задача:

> Найди, к какому порту сети подключена конкретная камера.

Вместо готовой функции `trace_camera()` плагин Network предоставляет универсальные операции:

* подключение к сетевому устройству;
* выполнение read-only команды;
* получение MAC-таблицы;
* ARP;
* LLDP/CDP;
* состояние интерфейса;
* VLAN;
* PoE;
* ping;
* SNMP-запросы.

Основной агент сам строит последовательность действий. Если задача сложная, он может делегировать отдельные части subagents.

---

## 1. Реестр моделей как инструмент, доступный агенту

В SPLA должен существовать реестр доступных LLM- и других AI-моделей.

Сам агент может запросить этот реестр:

```text
list_models()
```

И получить структурированное описание:

```yaml
- id: qwen-local-network
  type: llm
  connection: lmstudio-local
  description: >
    Локальная модель. Хорошо разбирает короткие сетевые выводы
    и выполняет простые диагностические задачи.
    Хуже справляется с длинным планированием и большим числом инструментов.
  strengths:
    - network-output-analysis
    - classification
    - short-diagnostics
  weaknesses:
    - long-agent-loops
    - complex-planning
    - recovery-after-errors
  privacy: local
  cost: free
  speed: fast

- id: claude-main
  type: llm
  connection: anthropic-main
  description: >
    Сильна в длинном планировании, работе с большим контекстом,
    анализе неоднозначных результатов и восстановлении после ошибок.
  strengths:
    - complex-planning
    - tool-orchestration
    - architecture
  weaknesses:
    - external-cloud
    - higher-cost
  privacy: cloud
  cost: high
  speed: medium
```

На основании задачи агент сам выбирает модель для subagent:

```text
run_subagent(
    model = "qwen-local-network",
    task = "Разбери выводы LLDP и MAC-таблиц",
    tools = [...],
    context = [...],
    limits = ...
)
```

Или:

```text
run_subagent(
    model = "claude-main",
    task = "Построй дальнейший план диагностики по собранным данным",
    tools = [...],
    context = [...],
    limits = ...
)
```

То есть реестр моделей — это часть рабочего мира агента, такая же доступная ему сущность, как список SQL-подключений, сетевых устройств или браузерных профилей.

Это касается не только LLM:

* vision-моделей;
* OCR;
* speech-to-text;
* text-to-speech;
* генерации изображений;
* embedding-моделей;
* reranker-моделей.

Агент должен понимать, какая модель для чего предназначена, и выбирать её по описанию, ограничениям и политике.

## 2. Автоматическое сравнение моделей

Это отдельная подсистема.

Она не участвует в обычном выборе модели агентом, а запускает одну и ту же задачу на нескольких моделях автоматически.

Например:

```text
compare_models(
    task = "Найди порт подключения камеры CAM-17",
    models = [
        "claude-main",
        "gpt-main",
        "qwen-local-network",
        "gemma-local"
    ],
    runs_per_model = 5
)
```

Для каждого запуска SPLA фиксирует:

* исходную задачу;
* prompt;
* контекст;
* документацию;
* доступные инструменты;
* политики;
* версии плагинов;
* используемые подключения;
* состояние окружения;
* вызовы инструментов;
* промежуточные ошибки;
* итоговый ответ;
* время;
* стоимость;
* вмешательства пользователя.

Дальше автоматически сравниваются:

* успешность выполнения;
* точность результата;
* подтверждённость выводов;
* эффективность использования инструментов;
* количество лишних действий;
* ошибки;
* соблюдение политик;
* восстановление после ошибок;
* повторяемость;
* время;
* стоимость.

Пример результата:

| Модель      | Успех | Подтверждённость | Вызовы | Ошибки | Время |     Цена |
| ----------- | ----: | ---------------: | -----: | -----: | ----: | -------: |
| Claude      |   5/5 |              96% |   12.4 |    0.2 |  37 с |    $0.43 |
| GPT         |   4/5 |              91% |   10.1 |    0.6 |  31 с |    $0.29 |
| Qwen local  |   3/5 |              78% |   17.8 |    1.7 |  54 с | локально |
| Gemma local |   2/5 |              65% |   21.3 |    2.9 |  73 с | локально |

Эта система нужна, чтобы объективно выяснять:

* какая модель лучше для Network;
* какая лучше для SQL;
* какой prompt работает;
* сколько документации требуется;
* где локальной модели достаточно;
* где нужна сильная облачная модель;
* какая модель чаще нарушает политику;
* какой набор инструментов помогает или мешает.

---

## Главное различие

**Реестр моделей** нужен самому агенту для выбора исполнителя под конкретную подзадачу.

**Автоматическое сравнение моделей** нужно разработчику SPLA для оценки качества моделей, промптов, инструментов и конфигураций.

Это две независимые, но связанные возможности.

Кратко:

> Агент использует реестр моделей, чтобы выбирать subagents.
> SPLA использует evaluation subsystem, чтобы автоматически сравнивать эти модели на одинаковых задачах.
